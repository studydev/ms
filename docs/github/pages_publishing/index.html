저장 즉시 전 세계 공개

1

저장소 만들고 Pages 켜기

GitHub에 로그인한 뒤, 브라우저에서 아래 순서로 클릭만 하면 끝입니다. GitHub 계정이 없다면 먼저 무료 가입하세요.

  1. 우측 상단 ＋ → New repository
  2. 저장소 이름 입력 (예: my-docs), Public 선택, Add a README 체크, Create repository
  3. 저장소 화면에서 Settings → Pages 이동
  4. Source: Deploy from a branch → Branch는 main, 폴더는 /(root) 또는 /docs 선택 후 Save
완료 확인 Settings → Pages 상단에 "Your site is live at …" 메시지가 뜨면 성공. URL은 https://<계정>.github.io/<저장소>/ 형식입니다.
폴더는 /docs 추천 웹사이트 파일과 다른 자료를 분리할 수 있어 관리가 편합니다. Jekyll 같은 빌드 도구를 쓰지 않을 거라면, 해당 폴더에 .nojekyll이라는 빈 파일을 하나 두면 안전합니다.
2

HTML 파일 드래그 & 드롭으로 업로드

저장소 페이지(루트 또는 docs/ 폴더 안)에서 진행합니다.

  1. 상단 Add file → Upload files 클릭
  2. HTML 파일과 필요한 리소스(images/ 등)를 화면으로 드래그 & 드롭
  3. 하단 Commit changes 버튼 클릭
파일 이름은 index.html 폴더에 접근하면 자동으로 index.html이 열립니다. 여러 문서를 올릴 땐 문서별 하위 폴더를 만들고 각각 index.html을 두세요. 예: docs/report/index.html → URL .../report/.
3

URL 열어 확인하고 공유

커밋 후 1~2분(최대 10분) 안에 배포가 끝납니다. Settings → Pages에 안내된 URL로 접속해 보세요.

  • 잘 보이면 그 링크를 Teams · Outlook · 카카오톡 등에 그대로 붙여 공유하면 끝.
  • 수정이 필요할 땐 저장소에서 파일을 열어 연필 아이콘으로 편집 후 Commit — 즉시 반영됩니다.
안 보일 때 체크 3가지
  • Pages Source 폴더 설정이 실제 업로드한 폴더와 같은가
  • 파일 이름이 index.html인가 (대소문자 주의)
  • 배포 완료 전일 수 있음 — 1~2분 기다린 뒤 새로고침

실습으로 더 깊이 익히기

GitHub에서 Pages를 만들고 활용하는 가이드가 필요하다면, 아래 실습을 통해서 더 자세한 사용 방법을 익힐 수 있습니다. GitHub Skills가 제공하는 한국어 공식 핸즈온 코스입니다.

skills-kr/github-pages — GitHub Pages 실습 (한국어)

내 저장소에서 단계별 이슈를 따라가며 GitHub Pages를 직접 설정하고 배포해 보는 공식 실습입니다.

고급 · 개발자용 (선택)

필요한 경우에만 펼쳐 보세요. 기본 공유에는 필요 없습니다.

  1. 도메인 등록업체의 DNS 관리에서 CNAME 레코드 추가 — Host: docs, Value: <계정>.github.io
  2. 저장소 Settings → Pages → Custom domain에 docs.example.com 입력 → Save
  3. DNS 체크가 녹색이 되면 Enforce HTTPS 체크

루트 도메인(example.com)을 연결하려면 A/ALIAS/ANAME 레코드가 필요합니다. 최신 IP와 자세한 내용은 GitHub 공식 문서를 참고하세요.

터미널
$ git clone https://github.com/<계정>/<저장소>.git $ cd <저장소> && cp ~/report.html docs/index.html $ git add . && git commit -m "publish" && git push

push 즉시 자동 빌드·배포가 실행됩니다.

여러 문서를 올릴 때 랜딩 페이지에서 목록을 자동으로 렌더링하려면, docs/manifest.json을 Actions로 생성하는 방식이 편합니다. 이 사이트의 예시는 scripts/generate_manifest.py를 참고하세요.

꼭 알아둘 제한 사항

GitHub Pages는 비상업적 · 정적 용도의 무료 서비스입니다. 자세한 내용은 GitHub Pages 제한 문서에서 확인하세요.

항목기준
저장소 / 게시 사이트 크기각 1 GB 권장
월 대역폭100 GB (soft)
빌드 횟수시간당 10회 (custom Actions 사용 시 미적용)
배포 타임아웃10분 초과 시 자동 중단
상업 · 전자상거래 · SaaS 용도사용 금지
참고 GitHub Free 플랜은 공개 저장소에서만 Pages를 무료로 제공합니다. 비공개(Private) 저장소에 쓰려면 Pro/Team/Enterprise 플랜이 필요합니다. 또한 동적 백엔드(DB·API 서버)는 지원하지 않습니다.
