이제부터 나도 이커머스 상품 포토 디자이너!
스마트폰으로 찍은 한 장을 — 룩북 컷과 정면·측면·후면 스튜디오 컷으로

Azure OpenAI gpt-5.4가 상품의 디테일을 집요하게 관찰하고, gpt-image-2가 광고급 이미지로 다시 그려줍니다. 별도의 스튜디오, 모델 섭외, 후보정 작업 없이 “상품 사진 1장 → 룩북 + 360° 스튜디오 컷”을 자동화하는 방법을 단계별로 정리했습니다.

2026년 5월 기준 gpt-5.4 + gpt-image-2 이커머스 셀러 친화

셀러의 진짜 고민에서 출발합니다

상품은 좋은데 사진이 발목을 잡습니다. 매번 스튜디오를 빌리고, 모델을 섭외하고, 보정까지 외주로 돌리는 건 1인 셀러·스몰브랜드에겐 큰 부담입니다. 이 가이드는 “이미 가지고 있는 상품 사진 1장”을 출발점으로, AI가 같은 상품을 광고급 이미지로 다시 만들어주도록 설계합니다.

지금까지의 비용 구조
  • 스튜디오 대관 + 조명 세팅
  • 모델 섭외 + 메이크업·스타일링
  • 촬영 1회당 수십~수백 컷, 그중 사용 가능한 컷은 일부
  • 후보정 외주 비용과 대기 시간
이 가이드가 약속하는 것
  • 가지고 있는 상품 사진 1장만 있으면 시작
  • 같은 상품을 룩북 컷 + 정면·측면·후면 스튜디오 컷으로 자동 생성
  • 상품 디테일(색상·배색·단추·로고 등)은 AI가 집요하게 보존
  • 모드 파라미터 하나만 바꿔서 다양한 광고 소재 확보
핵심 인사이트 — 좋은 결과의 비결은 “좋은 모델”이 아니라, 모델에게 상품을 어떻게 설명해 주느냐입니다. gpt-5.4는 관찰자, gpt-image-2는 사진작가, 셀러는 디렉터 역할을 맡습니다.

전체 워크플로우

한 장의 입력 이미지가 광고용 결과물로 변환되는 과정을 5단계로 정리합니다. 각 단계는 다음 단계의 입력으로 흐릅니다.

입력 이미지

스마트폰·기존 촬영본 1장

상품 분석

gpt-5.4 멀티모달

프롬프트 산출

Output_Prompt 검수

샷 모드 생성

gpt-image-2 · 4가지 모드

광고 소재화

상세페이지 · SNS · 광고

분리된 책임
  • 분석 단계 (gpt-5.4) — “어떤 상품인가”를 텍스트로 정확히 기술
  • 검수 단계 (사람) — 누락된 디테일·잘못된 좌우 해석을 빠르게 보정
  • 생성 단계 (gpt-image-2) — 검수된 프롬프트와 원본 이미지를 함께 받아 광고급 컷 생성

사용 기술 스택

Azure OpenAI 안에서 분석 모델과 이미지 생성 모델을 따로 배포하고, 인증은 Entra ID 토큰 기반으로 통일합니다.

Azure OpenAI · gpt-5.4 멀티모달 입력을 받아 이미지를 부위별로 관찰하고 한국어로 상세 프롬프트를 작성합니다. 좌우 비대칭, 색상 순서, 부속 요소까지 텍스트화합니다.
Azure OpenAI · gpt-image-2 텍스트 프롬프트와 원본 참조 이미지를 함께 받아 1024×1024 광고급 이미지를 생성합니다. 룩북·스튜디오 모드를 모두 한 모델에서 처리합니다.
Microsoft Entra ID API Key 대신 DefaultAzureCredential로 토큰을 발급받아 두 모델 모두에 사용합니다. 키 노출 위험을 줄이고 권한은 RBAC로 관리합니다.
비용 메모 — 분석은 토큰 기반(이미지 토큰 포함), 이미지 생성은 1장당 정액 과금입니다. 같은 상품에 대해 분석 1회 + 4개 모드 생성으로 운용하면 컷당 비용이 일반 촬영 대비 큰 폭으로 낮아집니다.

  • 오픈마켓 대표 이미지
  • 옵션·컬러칩 옆 썸네일

후면 컷

등판·뒷판 디테일, 라벨·로고·뒷주머니 등 “뒤에서만 보이는” 디자인 포인트를 보여주는 컷입니다.

  • 디테일 컷 카탈로그
  • ‘뒤태’ 강조 광고용
1

환경 준비 — Azure OpenAI 배포와 인증

Azure OpenAI 리소스에서 gpt-5.4(분석)와 gpt-image-2(이미지 생성) 두 모델을 각각 배포합니다. 인증은 API Key 대신 Entra ID 기반 DefaultAzureCredential로 통일합니다.

사전 체크리스트
  • Azure 구독 + Azure OpenAI 리소스 (Sweden Central / East US 등 모델 가용 리전)
  • 두 모델 배포: gpt-5.4, gpt-image-2
  • 로컬에 az login 완료, 또는 Managed Identity 환경
  • Python 3.10+ 환경에 openai, azure-identity, python-dotenv, Pillow 설치
환경 변수 (.env 예시)
# Azure OpenAI 엔드포인트만 노출, 키는 사용하지 않음
AZURE_OPENAI_ENDPOINT=https://<your-resource>.cognitiveservices.azure.com/
AZURE_OPENAI_API_VERSION=2025-04-01-preview

# 분석 모델과 이미지 모델 배포 이름
AZURE_OPENAI_ANALYSIS_DEPLOYMENT=gpt-5.4
AZURE_OPENAI_IMAGE_DEPLOYMENT=gpt-image-2
공통 클라이언트 빌더
from azure.identity import DefaultAzureCredential, get_bearer_token_provider
from openai import AzureOpenAI

def build_client(endpoint: str, api_version: str) -> AzureOpenAI:
    token_provider = get_bearer_token_provider(
        DefaultAzureCredential(),
        "https://cognitiveservices.azure.com/.default",
    )
    return AzureOpenAI(
        api_version=api_version,
        azure_endpoint=endpoint,
        azure_ad_token_provider=token_provider,
    )

상품 분석 — gpt-5.4 멀티모달로 디테일 추출

입력 이미지를 gpt-5.4에 보내고, “이 상품을 처음 보는 디자이너”에게 설명한다는 관점으로 텍스트화합니다. 분석 결과는 다음 단계의 이미지 생성 프롬프트(Output_Prompt)로 사용합니다.

분석에서 반드시 잡는 항목
형태·실루엣

카테고리, 핏, 길이, 어깨선 등

색상

메인 컬러, 보조 컬러, 톤·채도

재질·텍스처

니트 조직, 직조 방향, 두께감

마감부

밑단, 커프스, 카라, 플래킷 배색

부속 요소

단추, 지퍼, 포켓, 패치, 로고

좌우 비대칭

이미지 기준 왼쪽/오른쪽 분리 기록

작은 디테일을 잡는 두 가지 트릭
  1. 전체 + 세부 크롭 동시 입력 — 원본 이미지 한 장에 더해, 좌/우 가장자리·소매 끝·밑단을 자른 크롭을 함께 보냅니다. 모델은 작은 띠나 색상 순서를 더 정확히 잡습니다.
  2. system / user 역할 분리 — 모델 역할·출력 계약은 system에 두고, 이번 작업의 분석 규칙은 마크다운 파일로 분리해 user에 넣습니다. 다른 상품에 그대로 재사용할 수 있습니다.
분석 호출 핵심 스니펫
# 1) 시스템 프롬프트는 모델의 고정 역할/출력 계약
SYSTEM_PROMPT = """
당신은 상품 디테일을 집요하게 관찰하는 시니어 패션 제품 디자이너이자 이미지 생성 프롬프트 작성 전문가입니다.

역할: 입력 이미지의 대표 상품 1개를 분석해, 이미지 생성 모델에 바로 넣을 수 있는 한국어 Output_Prompt를 작성합니다.
원칙:
- 보이지 않는 디테일은 만들지 않습니다.
- 좌/우, 상/하가 다르게 보이면 일반화하지 않고 분리해서 기술합니다.
- 작은 마감부와 배색은 detail crop을 근거로 full image와 교차 확인합니다.
출력 계약:
- '# Output_Prompt' 제목과 하나의 완성형 프롬프트만 출력합니다.
- 'crop' 같은 작업 메타 표현은 출력하지 않습니다.
"""

# 2) 사용자 프롬프트는 이번 작업의 분석 규격(MD 파일) + 입력 자료 설명
user_prompt = build_user_prompt(instructions_markdown, detail_note="")

# 3) 멀티모달 호출 — 전체 이미지 + 자동 세부 크롭들을 함께 전달
result = client.chat.completions.create(
    model="gpt-5.4",
    messages=[
        {"role": "system", "content": SYSTEM_PROMPT},
        {"role": "user", "content": [
            {"type": "text", "text": user_prompt},
            *build_image_content(image_path, include_crops=True),
        ]},
    ],
    max_completion_tokens=4096,
)
output_prompt = result.choices[0].message.content
Tip — 자동 크롭은 PIL로 좌/우 절반, 상/하 절반, 밑단 띠 영역 등 5~10개 정도를 1024×1024로 확대해서 함께 보냅니다. 작은 디테일 인식률이 눈에 띄게 좋아집니다.
3

분석 결과 검수 — 사람의 5분이 결과를 바꿉니다

gpt-5.4가 만든 Output_Prompt는 사람이 한 번 빠르게 읽어주는 것만으로 품질이 크게 올라갑니다. 다음 6가지만 체크하세요.

  1. 대표 상품이 1개로 좁혀졌는가 — 가방·휴대폰·다른 의류는 제거되었는지 확인
  2. 색상 순서와 개수 — 단추·지퍼·로고 등이 보이는 순서대로 적혔는지
  3. 좌우 비대칭 — 소매·밑단·옆선이 “이미지 기준 왼쪽/오른쪽”으로 분리되어 있는지
  4. 가려진 부위 — 손·팔·다른 옷에 가려진 부분이 “보이는 범위에서는”처럼 표현되어 있는지
  5. 일반화 표현 — “multiple”, “various”, “symmetric” 같은 뭉뚱그린 단어가 남아 있지 않은지
  6. 방향 고정 지시 — “이미지 기준 왼쪽/오른쪽을 유지하고 좌우 반전하지 말 것” 문구가 들어 있는지
5분 보정의 위력 — 모델이 한쪽 부위를 다르게 해석했을 때, 사람의 한 줄짜리 detail-note(예: “이미지 기준 왼쪽 소매 커프스에는 하늘색+검정 띠가 있다”)만 추가해도 결과가 크게 달라집니다.
Output_Prompt 형태 (예시 구조)
# Output_Prompt
1024x1024 정방형, 에디토리얼 룩북 스타일의 고품질 착용컷, 대표 상품은 [상품 카테고리/색상] 1개만 노출,
얼굴은 프레임 밖으로 완전히 크롭, 인물 1명, 프론트 뷰 기반에 상체 중심 9:1 구도, 상품이 화면의 약 80%.
... (실루엣·재질·핏)
... (단추·포켓·플래킷 등 부속 요소의 색상 순서·개수·위치)
... (이미지 기준 왼쪽 / 오른쪽 마감부 비대칭 보존)
... "이미지 기준 왼쪽/오른쪽을 유지하고 좌우 반전하지 말 것"

샷 모드 생성 — gpt-image-2 + 모드 파라미터

같은 Output_Prompt를 그대로 두고, 모드별 “스타일 헤더”를 앞에 붙여 4가지 광고 컷으로 분기합니다. 원본 입력 이미지는 항상 함께 첨부해 상품 디테일을 유지합니다.

모드별 스타일 헤더 (한국어 예시)
lookbook:
  "1024x1024 정방형, 자연광 룩북 착용컷, 모델은 얼굴 크롭, 자연스러운 비대칭 데일리 포즈,
   미니멀 라이트 그레이 스튜디오 배경, 상품이 화면의 약 80%를 차지."

front:
  "1024x1024 정방형, 순백색 #FFFFFF 배경의 단독 정면 스튜디오 컷, 상품만 노출,
   부드러운 그림자, 평면적이고 왜곡 없는 정중앙 정렬, 모델·소품 없음."

side:
  "1024x1024 정방형, 순백색 배경의 측면 스튜디오 컷, 상품을 90도 옆에서 본 ‘얼짱각’ 측면,
   두께감과 옆선 라인이 잘 드러나는 구도, 모델·소품 없음."

back:
  "1024x1024 정방형, 순백색 배경의 후면 스튜디오 컷, 상품의 뒷판이 정확히 정중앙에 보이도록 회전,
   라벨/뒷주머니/등판 디테일이 선명하게 드러나는 구도, 모델·소품 없음."
이미지 생성 호출 핵심 스니펫
# style_headers: dict[str, str] — 위에서 정의한 4가지 헤더
def render_image(
    client: AzureOpenAI,
    output_prompt: str,
    reference_image: Path,
    mode: str,
    output_path: Path,
) -> Path:
    style_header = style_headers[mode]
    final_prompt = f"{style_header}\n\n{output_prompt}"

    with open(reference_image, "rb") as img:
        result = client.images.edit(
            model="gpt-image-2",
            image=img,
            prompt=final_prompt,
            size="1024x1024",
            input_fidelity="high",
        )

    # Azure OpenAI는 b64_json 또는 url 응답 모두 지원
    payload = result.data[0]
    if payload.b64_json:
        output_path.write_bytes(base64.b64decode(payload.b64_json))
    else:
        output_path.write_bytes(download(payload.url))
    return output_path
CLI 사용 예 (한 번에 4컷)
$ python analyze_image.py --image input/sample.png --output output/sample.prompt.md
$ for mode in lookbook front side back; do
  python create_image.py --input-image input/sample.png \
              --prompt-file output/sample.prompt.md \
              --mode $mode \
              --output output/sample.$mode.png
$ done
좌우 반전 주의 — 이미지 생성 모델은 종종 좌우를 뒤집어 해석합니다. 모드와 무관하게 Output_Prompt 안에 “이미지 기준 왼쪽/오른쪽을 유지하고 좌우 반전하지 말 것”을 항상 포함하세요.
5

광고 소재로 활용 — 한 번 만들어 여러 채널에 재사용

네 가지 컷을 채널별로 다르게 조합하면 한 상품에 대해 다양한 광고 소재가 만들어집니다.

  • 상세페이지 — 상단 메인은 lookbook, 중간 “디테일 컷”은 front / side / back 3종 슬라이드
  • 오픈마켓 썸네일 — front 단독, 옵션 컬러칩 옆에 가장 잘 어울림
  • SNS 피드 / 릴스 표지 — lookbook + 캡션 텍스트 오버레이
  • 퍼포먼스 광고 A/B 테스트 — 룩북 vs 스튜디오 정면, 측면 vs 후면을 광고 소재로 비교
운영 팁 — 같은 상품이라도 시즌/프로모션마다 모드별 “스타일 헤더”의 톤(예: 자연광 → 야외 골든아워, 흰색 → 베이지 무드)만 바꿔 재실행하면 시즌 콘텐츠를 빠르게 만들 수 있습니다.

프롬프트 설계 원칙 — 한 번 잘 짜두면 재사용

하나의 시스템 프롬프트와 분석 규칙 마크다운을 잘 만들어두면, 상품이 바뀌어도 같은 품질을 유지할 수 있습니다.

system prompt — 잘 안 바뀌는 것
  • 모델의 고정 역할 (시니어 디자이너 + 프롬프트 작성자)
  • 출력 계약 (# Output_Prompt + 단일 프롬프트만)
  • 금지 사항 (없는 디테일 만들지 말 것 / 좌우 반전 금지)
  • 기준 좌표계 (이미지 기준 왼쪽·오른쪽으로 고정)
user prompt + MD — 매번 바뀌는 것
  • 이번 입력 구성 (full image, detail crops, detail-note)
  • 이번 작업의 분석 규격과 카테고리별 규칙
  • (선택) 사람이 검수해 보정한 핵심 디테일 노트
  • (선택) 채널별 톤 변경 (자연광 vs 골든아워 등)
결과 안전 장치 — Output_Prompt에는 항상 “반드시 유지”, “정확히 보존”, “일반화하지 말 것”, “좌우 반전하지 말 것” 같은 보존 키워드를 강제로 포함시키세요. 이미지 생성 모델이 임의 해석할 여지를 줄여줍니다.

자주 빠지는 함정 4가지

1. 좌우 반전

생성 모델이 ‘착용자 기준’으로 좌우를 바꿔버리는 경우가 많습니다. 분석 단계부터 “이미지 기준”으로 고정하고, 최종 프롬프트에 명시적인 반전 금지 문구를 넣으세요.

2. 밑단·디테일 누락

하의에 가려지거나 크롭으로 잘려 밑단 색상이 사라지는 경우가 흔합니다. 프롬프트에 “밑단 전체가 프레임 안에 들어오게”라는 구도 지시와 색상 순서를 함께 명시하세요.

3. 비상품 요소 보존

원본에 있는 휴대폰·가방·로고 티셔츠 같은 비상품 요소가 결과에 남는 경우가 있습니다. 분석 단계에서 “대표 상품 1개만 노출”을 고정 규칙으로 넣으세요.

4. 과도한 일반화

“대칭 스트라이프”, “여러 색의 단추”처럼 뭉뚱그리면 결과가 크게 달라집니다. 색상 순서·개수·상대 폭(예: 검정(1)+흰색(5)+검정(1))까지 적도록 분석 규칙을 강제하세요.
