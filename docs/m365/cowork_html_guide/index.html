실전 가이드

M365 Cowork로
HTML 문서 & 슬라이드 만들기
대화 한 번으로 시각적 결과물을, Git Push 한 번으로 전 세계에 공유

짧은 시간 안에 시각적으로 효과적인 기술 문서, 가이드, 프레젠테이션 슬라이드를 만들고 싶으신가요?
M365 Cowork에게 말하면 됩니다. 나머지는 이 가이드가 알려드립니다.

읽기 시간: 10분 2026년 4월 17일 한국어

이런 결과물을 만들 수 있습니다

아래는 M365 Cowork에 대화형으로 요청하여 실제로 만들어낸 결과물 유형입니다. 스타일, 레이아웃, 코드 하이라이트까지 AI가 처리합니다.

기술 가이드 문서

문서형 HTML

개념 설명, 코드 블록, 표, 하이라이트 박스가 포함된 완성도 높은 기술 문서를 대화 한 번으로 생성합니다.

프롬프트 예시 "Azure Cosmos DB 파티션 키 선택 가이드를 한국어로 정리해줘. 핵심 개념, 모범 사례, 안티 패턴을 표로 정리하고 코드도 포함해서 HTML 파일로 만들어줘."

프레젠테이션 슬라이드

슬라이드형 HTML

키노트 스타일의 슬라이드를 HTML로 생성합니다. 페이지 넘김, 시각적 레이아웃, 아이콘까지 AI가 디자인합니다.

프롬프트 예시 "GitHub Actions CI/CD 소개를 10장 HTML 슬라이드로 만들어줘. 각 슬라이드마다 제목, 핵심 포인트 3개, 관련 아이콘을 넣어줘."

비교 분석 & 요약

정리형 HTML

여러 기술·서비스를 비교하는 표, 장단점 카드, 의사결정 플로우차트를 시각적으로 구성합니다.

프롬프트 예시 "Azure App Service vs Azure Functions vs AKS를 비교하는 HTML 문서를 만들어줘. 표와 의사결정 흐름도를 포함하고 한국어로."

핵심: 예전에는 HTML 파일을 만들려면 CSS, JavaScript를 직접 작성해야 했지만, M365 Cowork에게 원하는 형태를 설명하면 하이라이트, 반응형 레이아웃, 코드 블록까지 갖춘 문서를 몇 분 안에 받을 수 있습니다.

왜 이 방법인가

Cowork로 만든 멋진 HTML, 그런데 어떻게 다른 사람에게 보여주나요?

기존의 고충
  • HTML 파일을 만들어도 공유할 곳이 마땅치 않음
  • 웹서버를 직접 구축·운영하려면 비용과 노력이 큼
  • 파일을 메일로 보내면 레이아웃이 깨지거나 열기 불편
  • 문서가 늘어날수록 관리·디자인 일관성 유지가 어려움
  • 업데이트할 때마다 파일을 다시 보내야 함
이 워크플로의 해결
  • GitHub Pages로 무료 정적 호스팅
  • git push 한 번으로 자동 배포
  • 공용 CSS/JS 템플릿으로 일관된 디자인
  • manifest.json 기반 자동 목록 관리
  • 커스텀 도메인으로 깔끔한 URL 공유

1

Cowork에서 HTML 문서 생성하기

M365 Cowork(cowork.microsoft.com)에서 원하는 주제를 대화형으로 요청합니다.

사례별 프롬프트 패턴
기술 문서를 만들 때

"[주제]에 대해 한국어로 정리해줘. 핵심 개념, 모범 사례, 예시 코드를 포함하고, 코드 블록에 구문 하이라이트를 넣어서 HTML 파일로 만들어줘."

표로 정리 코드 예시 포함 목차 추가
슬라이드를 만들 때

"[주제]를 10장 분량의 HTML 슬라이드로 만들어줘. 각 슬라이드에 제목, 핵심 포인트 3개, 아이콘을 넣고 키보드 좌우 화살표로 넘길 수 있게 해줘."

슬라이드 형태 페이지 넘김 다크 테마
단계적 고도화 전략

한 번에 완벽한 결과를 기대하기보다, 대화를 이어가며 점진적으로 품질을 높이세요.

1차: 콘텐츠 생성
"[주제]에 대해 HTML 문서로 정리해줘" → 기본 구조와 콘텐츠 확보
2차: 시각적 개선
"스타일을 좀 더 세련되게 해줘. 카드 레이아웃과 아이콘을 추가해줘" → 디자인 강화
3차: 기능 추가
"목차를 추가하고 반응형으로 바꿔줘. 코드 복사 버튼도 넣어줘" → 완성도 향상
4차: 슬라이드 변환 (선택)
"이 내용을 기반으로 PowerPoint 슬라이드로 변환해줘" → 프레젠테이션용
Cowork의 HTML 출력 Cowork는 대화 중 생성한 HTML을 로컬 파일로 저장합니다. "HTML 파일로 만들어줘"라고 요청하면 독립 실행 가능한 HTML 파일이 세션 출력 폴더에 저장됩니다.
2

출력 파일 위치 찾기

Cowork가 생성한 HTML 파일은 로컬 파일 시스템에 자동 저장됩니다.

파일 저장 경로 (Windows)
Windows 경로
C:\Users\<사용자명>\Documents\Coworker\sessions\<세션-ID>\output\ # 예시: C:\Users\hyounsoo\Documents\Coworker\sessions\a1b2c3d4-e5f6-7890\output\report.html
폴더 구조
📁 Documents └── 📁 Coworker └── 📁 sessions ├── 📁 a1b2c3d4-e5f6-7890 │ └── 📁 output │ ├── 📄 report.html │ ├── 📄 slides.html │ └── 📄 summary.html └── 📁 f9e8d7c6-b5a4-3210 └── 📁 output └── 📄 guide.html
세션 ID 확인 각 Cowork 세션마다 고유 세션 ID 폴더가 생성됩니다. 가장 최근 수정된 폴더가 마지막으로 작업한 세션입니다. 탐색기에서 수정한 날짜 기준으로 정렬하면 빠르게 찾을 수 있습니다.
3

나만의 GitHub Pages 만들기 & 파일 넣기

Cowork에서 만든 HTML을 공유하려면 호스팅할 곳이 필요합니다. GitHub Pages를 사용하면 무료로 정적 사이트를 만들 수 있습니다.

3-1. 나만의 GitHub Pages 저장소 만들기

이미 만들어진 템플릿(studydev/ms 등)을 Fork하거나, 새 저장소를 만들어 GitHub Pages를 활성화합니다.

1. GitHub에서 새 저장소 생성 (또는 Fork)
github.com/new에서 저장소를 만들거나, 기존 템플릿 저장소를 Fork합니다.
2. GitHub Pages 활성화
저장소 Settings → Pages → Source에서 Deploy from a branch를 선택하고 main 브랜치의 /docs 폴더를 지정합니다.
3. 로컬에 Clone
생성한 저장소를 로컬에 Clone합니다. 이후 VS Code나 파일 탐색기에서 작업할 수 있습니다.
이미 저장소가 있다면 이 단계는 건너뛰세요. 기존 저장소를 VS Code로 열면 바로 다음 단계로 진행할 수 있습니다.
3-2. 파일 탐색기에서 HTML 파일 복사

복잡한 CLI 명령은 필요 없습니다. 파일 탐색기(또는 VS Code Explorer)에서 직관적으로 작업합니다.

1. 카테고리 폴더로 이동
저장소의 docs/ 아래에서 Azure · M365 · GitHub 중 해당 카테고리 폴더를 엽니다.
2. 새 폴더 만들기
카테고리 폴더 안에 문서용 하위 폴더를 만듭니다 (예: cosmos_db_partition).
3. HTML 파일 복사
Step 2에서 찾은 Cowork 출력 HTML 파일을 새로 만든 폴더에 드래그 & 드롭합니다.
VS Code에서 작업하기 저장소를 VS Code로 열면 좌측 Explorer에서 폴더 생성, 파일 드래그 & 드롭이 모두 가능합니다. CLI 없이도 직관적으로 작업할 수 있습니다.
4

GitHub Copilot으로 디자인 변환하기

파일을 복사해 넣었다면, VS Code에서 GitHub Copilot에게 사이트 표준에 맞게 변환해 달라고 요청하면 됩니다. Copilot이 인라인 스타일 제거, 공용 CSS/JS 적용, 사이드바·네비게이션 추가를 자동으로 처리합니다.

Copilot에게 요청하는 방법

VS Code에서 저장소를 열고 Copilot Chat에 아래와 같이 요청합니다.

Copilot 프롬프트 예시
"docs/azure/cosmos_db_partition/ 폴더에 새로 추가된 HTML 파일을 이 사이트의 표준 디자인과 폴더 구조에 맞게 변환해줘. copilot-instructions.md 규칙을 따라서 공용 CSS/JS만 사용하고, 사이드바와 네비게이션을 포함한 index.html로 만들어줘."
Copilot이 자동으로 처리하는 것
인라인 스타일 제거

<style> 블록 → site.css 참조

인라인 스크립트 제거

<script> 블록 → site.js 참조

표준 레이아웃 적용

top-nav, sidebar, hero 구조

CSS 변수 활용

색상·폰트 하드코딩 대신 변수

핵심 포인트 이 저장소에는 copilot-instructions.md가 이미 설정되어 있어서, Copilot이 사이트의 디자인 규칙(공용 CSS/JS, 네비게이션 구조, 코드 블록 스타일 등)을 자동으로 인식하고 적용합니다.
5

manifest 등록 & 배포

사이트 목록에 등록하고 Git push로 배포합니다. 두 작업을 한번에 처리합니다.

5-1. manifest.json에 항목 추가
docs/manifest.json (해당 카테고리의 docs 배열에 추가)
{ "slug": "cosmos_db_partition", "title": "Azure Cosmos DB 파티션 키 선택 가이드", "description": "파티션 키 설계 모범 사례와 안티 패턴 정리", "path": "azure/cosmos_db_partition/" }
5-2. 커밋 & 배포
Git 커밋 & 배포
# 1. 변경 파일 스테이징 $ git add docs/azure/cosmos_db_partition/index.html docs/manifest.json # 2. Conventional Commits 형식으로 커밋 $ git commit -m "docs: Cosmos DB 파티션 키 가이드 추가" # 3. main 브랜치에 push → GitHub Pages 자동 배포 $ git push origin main

고급 활용 팁

Cowork로 먼저 HTML 슬라이드를 만들고, 그 구조를 기반으로 PowerPoint를 요청하면 일관된 디자인의 프레젠테이션을 얻을 수 있습니다.

  1. "이 주제를 10장 슬라이드로 정리해서 HTML로 만들어줘"
  2. HTML 슬라이드의 내용과 구조를 검토·수정
  3. "이 HTML 슬라이드를 기반으로 PowerPoint 파일로 변환해줘"
핵심 HTML 슬라이드에서 내용을 먼저 확정한 뒤 PPT로 변환하면, 콘텐츠 수정과 디자인 작업을 분리할 수 있어 훨씬 효율적입니다.

  1. 각 HTML 파일을 해당 카테고리 폴더에 original.html로 복사
  2. Copilot에게 폴더 목록을 전달하며 일괄 변환 요청
  3. manifest.json에 모든 항목 일괄 추가
  4. 한번의 커밋으로 모두 배포

  • Cowork에서 새 버전의 HTML을 생성
  • 기존 index.html이 있는 폴더에 original.html로 복사
  • Copilot에게 "기존 index.html의 콘텐츠를 original.html 기반으로 업데이트해줘"라고 요청
  • 레이아웃과 네비게이션은 유지하면서 콘텐츠만 교체됨

  • 문서 폴더 아래 images/ 하위 폴더를 만들어 이미지 파일을 함께 복사
  • HTML 내 이미지 경로를 상대 경로(images/screenshot.png)로 수정
  • 외부 URL 이미지는 가능하면 로컬로 다운로드하여 안정성 확보

더 높은 품질의 HTML을 얻기 위한 프롬프트 기법:

레퍼런스 제공 "이 MS Learn 문서를 참고해서 정리해줘: [URL]" — 공식 문서 URL을 첨부하면 정확도가 높아집니다.
스타일 지정 "Bootstrap 5 기반으로 카드 레이아웃을 사용해줘. 색상은 #0078D4 계열로." — 구체적인 디자인 요구사항을 명시합니다.
